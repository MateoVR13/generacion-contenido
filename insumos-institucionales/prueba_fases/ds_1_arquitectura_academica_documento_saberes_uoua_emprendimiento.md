## Contenido desarrollado

### 1. Titulo academico del documento

**Titulo principal propuesto:**  
`Validacion temprana de oportunidades de emprendimiento: de problema real a decision Gate 1`

**Subtitulo institucional propuesto:**  
`Documento de Saberes para Emprendimiento 1 / Fundamentos de Emprendimiento`

**Linea de identificacion:**  
Centro de Emprendimiento e Innovacion Sostenible - CEIS, Universidad de America.

### 2. Proposito del documento

El Documento de Saberes orientara al estudiante en la comprension y aplicacion de los conceptos, procedimientos, criterios eticos y herramientas necesarias para validar una oportunidad inicial de emprendimiento cientifico, tecnologico, creativo, social o sostenible. Su funcion no sera presentar teoria general de emprendimiento, sino acompanar una ruta aplicada de Aprendizaje Basado en Retos que conduzca a un portafolio de evidencias y sustentacion Gate 1.

El documento debe permitir que el estudiante:

1. Diferencie problema, oportunidad y solucion inicial.
2. Caracterice usuarios, clientes, empresas o comunidades vinculadas con un problema real.
3. Formule supuestos e hipotesis verificables.
4. Disene instrumentos exploratorios de validacion temprana.
5. Registre evidencia positiva y negativa de manera trazable.
6. Construya un prototipo de baja fidelidad coherente con el problema y los hallazgos.
7. Analice hallazgos y tome una decision Gate 1: continuar, pivotar, reformular o cerrar.
8. Comunique el proceso con claridad, criterio etico y uso responsable de tecnologia.

### 8. Politica de citacion APA 7

| Regla | Aplicacion |
|---|---|
| Cita obligatoria | Toda definicion especializada, modelo, teoria, afirmacion derivada de fuente, marco normativo o clasificacion disciplinar debe llevar cita. |
| Parafrasis academica | Se priorizara la parafrasis con cita narrativa o parentetica. No se copiaran fragmentos extensos. |
| Citas textuales | Se evitaran salvo necesidad academica; si se usan, deben ser breves y con referencia verificable. |
| No invencion | No se inventaran paginas, DOI, enlaces, ediciones, editoriales, fechas ni autores. |
| Fuentes por entrega | Cada subfase incluira solo las referencias efectivamente usadas en esa entrega. |
| Fuentes base | Osterwalder et al. (2014), Ries (2011), Brown (2009), Blank y Dorf (2012) se usaran como base conceptual cuando el contenido las requiera. |
| Fuentes normativas | Politicas institucionales, lineamientos o normas legales solo se citaran si se cuenta con identificacion verificable. |
| PDF local | El PDF local de `The Lean Startup` no se distribuira ni se citara por pagina hasta validar derechos y condiciones de uso. |
| Referencias consolidadas | DS-9 integrara la lista unica de referencias APA 7. |

### 9. Criterios de rigor cientifico y academico

| Criterio | Verificacion en el documento |
|---|---|
| Trazabilidad curricular | Cada capitulo debe conectarse con el RA, los saberes y el portafolio Gate 1. |
| Precision conceptual | Los conceptos deben diferenciarse sin ambiguedad: problema, oportunidad, solucion, hipotesis, evidencia, prototipo y decision. |
| Sustento bibliografico | Las ideas teoricas y modelos deben apoyarse en fuentes base o fuentes autorizadas. |
| Aplicabilidad | Cada concepto central debe traducirse en ejemplos y actividades de apropiacion. |
| Balance teoria-practica | La teoria debe explicar decisiones aplicadas, no funcionar como contenido aislado. |
| Reconocimiento de limites | El documento debe advertir que la evidencia temprana no equivale a prueba definitiva de mercado o impacto. |
| Etica transversal | La interaccion con personas, empresas o comunidades debe cuidar consentimiento, datos, lenguaje y confidencialidad. |
| No ampliacion indebida | No se incorporaran contenidos avanzados de plan financiero, constitucion empresarial o aceleracion si no aportan al RA. |
| Transparencia evaluativa | Cada actividad debe indicar que evidencia produce y que criterio de rubrica fortalece. |
| Claridad editorial | El lenguaje debe ser academico, directo, comprensible para pregrado y sin inflacion conceptual. |

## Criterios eticos/legales

| Criterio | Decision arquitectonica |
|---|---|
| Interaccion con actores | Debe aparecer desde la ruta inicial y repetirse en cada momento donde se recolecte informacion. |
| Consentimiento | Cada instrumento exploratorio debe incluir orientacion para explicar finalidad academica y solicitar autorizacion. |
| Datos personales | El documento debe recomendar no recolectar datos sensibles innecesarios ni publicar informacion identificable. |
| Comunidades y empresas | Los ejemplos deben evitar estigmatizacion y proteger informacion comercial, comunitaria o institucional sensible. |
| Uso de IA y tecnologia | El documento debe exigir verificacion, citacion de apoyos y criterio propio. |
| Derechos de autor | No se deben reproducir libros, PDFs o materiales no autorizados; se usaran referencias y enlaces permitidos. |
